Adaptive Fitness Coach / Phone

The phone is for setup and review only — never used during a workout. Its core job is hands-on: the user picks their days, builds their own routines, and fills each session with workouts from a library — arranged as cards, each with a form animation and a one-line "what it's good for." The AI is a secondary shortcut: if someone doesn't know what to build, it drafts a week in plain language that drops straight into the same editor.

Behavior
  • Each workout is a card showing: form animation (loops), name, 1–2 sentence explanation, a "Good for" tag, and the seed sets × reps · weight.
  • Drag the ≡ handle to reorder. Order is the literal play order on the watch (cards B1/B2). Swipe a card to remove; + Add returns to the library (P3).
  • Tapping a card opens its editor: change sets/reps, adjust the seed weight within the user's dumbbell range, replace the exercise. The weight is a seed — set once, then auto-tuned; the user won't return to edit it (N1/N7).
  • Card numbering + the "this is what your watch shows" line make the phone↔watch mapping explicit and build trust.
Data / API
  • Ordered list of {exerciseId, sets, reps, seedWeight} on the routine record. Order index = watch sequence index.
  • Animation + copy resolved from the library by id (not duplicated onto the routine).
  • Saved → synced to watch via WatchConnectivity; pairs with watch B1 / B2.

AI draft — a shortcut, not the default

For the user who doesn't know what to build. They describe what they want in plain language; the AI assembles a week from the same library and drops it into the same editor — every card still reviewable, reorderable, and removable. The AI never produces anything the user can't then edit by hand.

‹
Draft with AISHORTCUT
Describe it —
we'll draft a week.
Get stronger at home with dumbbells, 3 days a week. Nothing too hard on my knees.
You'll review every workout afterward — reorder, swap, or remove anything before you start.
Behavior
  • Reached only from the de-emphasized link on P1 — never blocks or front-runs the manual flow.
  • Free-text intent → a drafted week: routines, days, and exercises selected from the existing library (not invented), with seed weights/reps.
  • Output lands in the normal builder (P2→P4 state, pre-filled). It is a starting point the user edits — never an opaque plan they must accept. Constraints like "easy on my knees" bias exercise selection.
Data / API
  • Intent → routine generation constrained to library exercise ids (model returns ids, not free-form moves).
  • Produces the same routine records the manual flow does — fully editable, no special-casing downstream.
